# Architecture and Database Schema Revisions

This document describes the changes made between the original architecture/schema and their revised versions.

## Architecture Changes

### Removed: Map View

The Map View was removed from the application architecture. The customer's primary interest is in the analytics generated by StoreFlow rather than spatial map navigation. Since all relevant spatial data (heatmaps, aisle zones, camera views) is already surfaced through dedicated analytics views, the Map View added complexity without delivering additional value. Removing it keeps the interface focused on the analytics workflow the customer relies on.

## Database Schema Changes

### Store

| Change | Field | Type | Description |
| --- | --- | --- | --- |
| Added | `height` | `int` | The height dimension of the store layout |
| Added | `width` | `int` | The width dimension of the store layout |

These fields allow the application to define the physical dimensions of a store, which is used for rendering heatmaps, positioning aisles, and running simulations against an accurately sized layout.

### Product

| Change | Field | Type | Description |
| --- | --- | --- | --- |
| Added | `order` | `int` | The display/sort order of the product within its aisle |

This field provides explicit control over how products are ordered within an aisle, supporting consistent product arrangement in analytics views and simulation rendering.

### Customer

| Change | Field | Type | Description |
| --- | --- | --- | --- |
| Added | `age` | `str` | The estimated age group of the customer |
| Added | `sex` | `str` | The estimated sex of the customer |

These demographic attributes enable customer-level analytics such as product preferences by age group, purchasing patterns by gender, and demographic breakdowns across aisles and revenue views.

### Camera

| Change | Field | Type | Description |
| --- | --- | --- | --- |
| Removed | `location_x` | `int` | The x-coordinate of the camera in the store |
| Removed | `location_y` | `int` | The y-coordinate of the camera in the store |
| Added | `relative_file_path` | `str` | The file path to the camera's video relative to the project |

The camera's physical coordinates were replaced with a reference to its video file. Spatial positioning of cameras is handled through the Camera Merge View and Zone Setup View instead of being stored as raw coordinates, while the file path allows the application to locate and load the associated video footage directly.
